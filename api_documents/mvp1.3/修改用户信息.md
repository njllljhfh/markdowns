# 修改用户信息



> 设计人：牛金龙



- 描述：修改用户信息

- 请求URL：/api/users/info/<user_id>

- 请求类型：PUT

- 请求参数

  ```json
  {
      "user_update_data":{ # 用户要更新的 数据的字典
      					"user_name":"张三", # 新的用户名
  			 			},
      "create_oper":"2018102439652891014", # 执行修改用户的操作人id(当前登录的用户)
      "group_id_new_ls":["2018102439652891012","2018102439652891016"], # 新的 用户和组的关系
  	"role_id_new_ls":[3,4],# 新的 用户和组的关系
      #"group_id_old_ls":["2018102439652891011","2018102439652891015"],#要删除的 用户和组的关系
  	#"role_id_old_ls":[1,2],# 要删除的 用户和角色关系
  }
  ```

- 说明

  ```
  1. 如果用户基本信息没有更新，则请求报文中没有 user_update_data 字段
  2. group_id_old_ls、role_id_old_ls,报文中暂时不用传这两个字段
  3. 请求url中的 <user_id> 是要进行修改的用户id
  ```

    "msg":"获取 用户id={self.user_id} 的权限信息成功",
    "uinfo_dict":{

                              "org_name":"研发部",  # 用户组织机构名称
      					},  
                  "group":{	# 用户组

      					},

                              "role_id":"1",  # 角色id
